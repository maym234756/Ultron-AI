Generated: 2026-07-03
Companion file: ULTRON_TOOL_BUILD_RANKING.txt

Goal:
Upgrade every current tool from useful prototype/utility level toward best-in-class reliability, observability, safety, and ergonomics.

Global quality bar for all tools:
1. Strong schemas: clear required fields, enums where useful, examples in descriptions, and backward-compatible aliases.
2. Smart targeting: tools should accept human-friendly inputs where possible, not only brittle low-level identifiers.
3. Structured results: return concise human text plus machine-readable details when the calling layer supports it.
4. Diagnostics: failures should include the attempted action, reason, current state, and likely next steps.
5. Safety gates: destructive actions need previews, confirmations, dry runs, or undo paths.
6. Observability: long-running and external tools need timing, status, and dependency checks.
7. Tests/checks: each module needs at least type-check coverage and focused smoke tests for critical paths.

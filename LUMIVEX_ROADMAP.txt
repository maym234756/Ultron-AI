================================================================================
                         LUMIVEX AI PRODUCT ROADMAP
                         Prepared: July 3, 2026

MISSION
--------------------------------------------------------------------------------
Make Lumivex AI better than cloud chat assistants by focusing on what cloud products
cannot do as well: local privacy, OS-level action, authenticated browser work,
connector workflows, self-upgrade, transparent routing, and controllable agent
execution.

Core positioning:

  ChatGPT can answer.
  Lumivex AI can answer, inspect, operate, verify, remember, connect, and improve
  itself locally.

--------------------------------------------------------------------------------
CURRENT STRENGTHS
--------------------------------------------------------------------------------
- Fully local Ollama-powered assistant
- 150 callable tools
- 52-service external connector registry
- Chat / Agent auto-router
- Auto intelligence profiles: Instant, Balanced, Deep, Research
- Health Command Center
- Model Compare
- Voice Conversation Mode
- Task Dashboard
- Prompt Enhance
- Self-Upgrade with guardrails
- RAG + long-term memory
- Browser, desktop, file, terminal, coding, and OS automation
- OpenAI-compatible /v1 API

PRIORITY 1 — ROUTER PREVIEW + DECISION CHIPS
--------------------------------------------------------------------------------
Status: Complete — built July 3, 2026.
Implemented:
- /api/router/preview endpoint for route previews.
- Route metadata saved on assistant messages.
- Decision chip showing Chat/Agent, intelligence profile, and confidence.
- Expandable reason, matched signals, and score breakdown.
- Override actions: Rerun as Agent, Rerun as Chat, Rerun as Deep,
  Rerun as Research, Verify with tools, and Make shorter.
- Validated with npm run build and focused editor diagnostics.

Goal:
Make Lumivex AI's automatic decisions visible, trustworthy, and overrideable.

Why it matters:
The prompt router is now one of Lumivex AI's most important intelligence layers. The
user should see why Lumivex AI chose Chat vs Agent and Instant vs Deep/Research.

Build:
- Add /api/router/preview endpoint.
- Return route decision:
  - Chat or Agent
  - Instant / Balanced / Deep / Research
  - confidence percentage
  - reason
  - matched signals
  - score breakdown: agent, chat, complexity, freshness
- Show a compact decision chip above or below each assistant message:
  - Chat · Instant · 98%
  - Agent · Research · 73%
- Add hover/click details: "Why Lumivex AI chose this mode".
- Add override buttons:
  - Rerun as Agent
  - Rerun as Chat
  - Rerun as Deep
  - Rerun as Research
  - Verify with tools
  - Make shorter

Expected impact:
- Higher trust
- Faster debugging of bad routing
- Better user control
- Stronger product feel than ChatGPT/Claude

PRIORITY 3 — AGENT TASK PLANNER
--------------------------------------------------------------------------------
Status: Complete — built July 3, 2026.
Implemented:
- Structured runtime task planner with goal, assumptions, steps, tools needed,
  verification method, done condition, task size, and tool budget.
- Agent task states streamed to the UI: Planning, Using tools, Verifying, Done,
  and Needs user input.
- Tool budgets by task size: Simple = 1, Normal = 3-6, Deep/Research = 10+.
- Failure-recovery rules added to the agent contract for browser selectors,
  command failures, missing files, and unavailable tools.
- Final answer contract strengthened: what changed, what was verified, what
  remains, and next best action.
- Agent trace UI now shows a collapsible task plan and live task-state badges.
- Validated with npm run build and an Agent SSE smoke test confirming planner
  and task-state events on the real Agent path.

Goal:
Make Agent Mode more disciplined, reliable, and easy to follow.

Why it matters:
Agent Mode should behave like a senior operator: plan, act, verify, and stop
when done. It should avoid wandering, repeated tool calls, and vague outcomes.

Build:
- Add a structured task planner object:
  - goal
  - assumptions
  - steps
  - tools needed
  - verification method
  - done condition
- Show task state in the UI:
  - Planning
  - Using tools
  - Verifying
  - Done
  - Needs user input
- Add tool budgets by task size:
  - Simple: 1 tool
  - Normal: 3-6 tools
  - Deep: 10+ tools
- Improve failure recovery:
  - Browser selector fails -> try text locator
  - Command fails -> parse error and repair
  - File missing -> search nearby paths
  - Tool unavailable -> explain fallback
- Strengthen final answer contract:
  - What changed
  - What was verified
  - What remains
  - Next best action

Expected impact:
- More reliable agent execution
- Less wasted time
- Better long-task transparency
- Stronger differentiation from cloud chat

PRIORITY 4 — CONNECTOR SETUP WIZARD
--------------------------------------------------------------------------------
Status: Complete — built July 3, 2026.
Implemented:
- Connector setup wizard embedded in the External Connectors panel.
- Non-secret local setup state stored outside the repo for preferred auth path,
  permission level, browser session readiness, API readiness, audit toggle,
  and latest connection test result.
- Permission levels: Read only, Draft changes, Apply with approval, and Fully
  automated for safe tasks only.
- Browser session setup flow with Open, Mark browser signed in, and Test
  connection controls.
- API/OAuth setup visibility through required environment variables without
  collecting or storing secrets in chat.
- Connector audit log for setup updates and connection tests.
- Backend endpoints: /api/connectors/:id/setup and /api/connectors/:id/test.
- Validated with npm run build and Salesforce setup/test endpoint smoke test.

Goal:
Turn the 52 connector registry from "browser-ready" into first-class workflows.

Why it matters:
Connectors are one of Lumivex AI's strongest ways to beat ChatGPT/Claude. Cloud chat
can describe Salesforce or Gmail; Lumivex AI can operate them locally through an
authenticated browser session or configured API token.

Build:
- Add connector setup wizard:
  - Browser session setup
  - OAuth/API token setup
  - Test connection
  - Permission level
  - Audit log status
- Connector permission levels:
  - Read only
  - Draft changes
  - Apply with approval
  - Fully automated for safe tasks only
- Per-connector workflows:
  - Salesforce: search leads, update activity, create task
  - Gmail: read inbox, summarize thread, draft reply, send after approval
  - Google Sheets: read rows, append row, update cell
  - YouTube: summarize video, search channel, review comments
  - Slack/Teams: summarize channels, draft messages
  - Stripe/PayPal/Square: review payments, draft billing actions
  - GitHub/Jira/Linear: review issues, summarize work, draft updates
- Connector audit log:
  - What Lumivex AI read
  - What Lumivex AI drafted
  - What Lumivex AI changed
  - When approval was requested

Expected impact:
- Converts integrations into real product value
- Adds enterprise-grade safety
- Creates a major moat versus cloud assistants

PRIORITY 6 — MEMORY 2.0
--------------------------------------------------------------------------------
Status: Complete — built July 3, 2026.
Implemented:
- Structured long-term memory entries with confidence, source, scope,
  expiration, and promotion metadata while preserving old JSONL memories.
- User, project, and temporary memory scopes.
- Temporary memory expiration filtering and promotion to permanent user or
  project memory.
- Conflict candidate detection for preference/deployment-style memories before
  saving new entries.
- Structured memory APIs that return review-ready entries while preserving the
  legacy text response for compatibility.
- Memory 2.0 review panel with scope summaries, filters, source/confidence
  badges, recent auto-saved memories, conflict warnings, delete, and promote
  actions.
- Validated with npm run build, Memory API create/list/promote/delete smoke
  test, router regression suite, and health check.

Goal:
Make memory more selective, organized, and trustworthy.

Why it matters:
Memory is powerful, but noisy memory makes answers worse. Lumivex AI should remember
important things and forget temporary or stale things.

Build:
- Add memory confidence and source.
- Add project-scoped memories.
- Add temporary memories with expiration.
- Add memory promotion:
  - "This seems important. Save permanently?"
- Add memory conflict detection:
  - "You previously preferred X, but now said Y. Update memory?"
- Add memory review panel:
  - what Lumivex AI remembers about the user
  - what Lumivex AI remembers about this project
  - recent auto-saved memories

Expected impact:
- Cleaner personalization
- Better project continuity
- Less stale context pollution

PRIORITY 8 — SPEED + LATENCY SYSTEM
--------------------------------------------------------------------------------
Goal:
Make Lumivex AI feel fast even when local models are slower than cloud inference.

Why it matters:
Speed is user experience. Local AI must avoid unnecessary model/tool work.

Build:
- Keep fast model warm.
- Cache model warmup status.
- Use fast model for:
  - titles
  - followups
  - prompt enhancement
  - short answers
  - router explanations
- Use primary model for:
  - Deep reasoning
  - Research synthesis
  - coding
  - agent tasks
- Add stream heartbeat for long tasks.
- Add visible task progress for Agent Mode.
- Add latency diagnostics in Health/Diagnostics panel.
- Avoid loading full agent prompt for knowledge-only or connector setup prompts.

Expected impact:
- Faster perceived response time
- Less waiting on simple prompts
- Better local-model experience

Status: COMPLETE
Completed build:
- Added observable model warmup status, fast-model readiness, and /api/latency/status diagnostics.
- Added Speed + latency system health row.
- Routed title generation, followups, and prompt enhancement through the detected fast model when available.
- Added SSE stream_status events for connected, waiting for first token, streaming, and done states.
- Rendered stream progress in the Agent Trace/loading UI.
- Validation: npm run build passed.

================================================================================
                         ULTRON - FULL CAPABILITIES
================================================================================

Ultron is a local AI assistant running on your Windows machine with full tool
access. Primary model: qwen2.5:14b. Vision: llava. Fast model: llama3.2:3b.
Embeddings: nomic-embed-text. Playwright-controlled Chromium for full browser
automation (headless:false). Passive screen observer runs in the background,
injecting live context. Real-time token streaming. Parallel tool execution.
Scored prompt router chooses Chat vs Agent and Instant/Balanced/Deep/Research.
Answer style controls tune output as Concise, Detailed, Technical, or Executive.
External Connector Hub tracks 52 supported services with typed native action
schemas and dry-run planning.

--------------------------------------------------------------------------------
AI INTELLIGENCE
--------------------------------------------------------------------------------
Ultron's reasoning uses a structured multi-step protocol:
  UNDERSTAND → DECOMPOSE → PLAN → ACT → VERIFY

Critical Thinking:
- Challenges assumptions; verifies claims with tools before acting
- Calibrated confidence: states "I know" vs "I believe" vs "I'm guessing"
- Shows reasoning chains: Because X → therefore Y → conclusion Z
- Self-checks before final answers: accurate? complete? best approach?
- Never confabulates — if unsure, says so and looks it up

Response Quality:
- Calibrates length to question complexity (no padding, no fluff)
- No sycophantic openers ("Certainly!", "Great question!", etc.)
- No hollow closers ("I hope this helps!", "Let me know if…")
- Uses markdown only when genuinely helpful
- Cites sources when using retrieved knowledge

Parallel Tool Execution:
- Can call multiple independent tools simultaneously in a single step
- Runs them with Promise.all for maximum speed
- E.g., search_web + read_file execute in parallel, not sequentially

Context Management:
- Context window: 8,192 tokens default (user-configurable 2,048–32,768)
- Auto-compresses long conversations (>18 turns) into a rolling summary
- Screen observer context injected into every request (configurable)
- RAG knowledge + long-term memories auto-recalled per query

Intelligence Profiles:
- Instant: shortest correct path, fast-model routing, minimal context overhead
- Balanced: default profile for normal chat and coding work
- Deep: stronger decomposition, edge-case checking, larger context floor
- Research: evidence-first synthesis with uncertainty handling and citations
- Profiles are persisted in browser settings and sent to both chat and agent APIs

Auto Router / Decision Engine:
- Scores every prompt before dispatch: agent score, chat score, complexity,
  freshness, confidence, and matched decision signals
- Auto-selects Chat vs Agent based on tool/action/connector/file/web needs
- Auto-selects intelligence mode based on speed, complexity, research, code,
  vision, and account-action cues
- Keeps simple knowledge prompts fast: "What is Salesforce?" -> Chat/Instant
- Routes action and research prompts correctly: "Open Gmail" -> Agent;
  "Research latest X with sources" -> Agent/Research
- Routes project, folder, codebase, and explicit local-path reviews to Agent/Deep
  so Ultron can request permission and use filesystem tools instead of guessing
- Settings toggles: Auto route Chat/Agent and Auto intelligence profile

Critical Thinking Upgrades:
- Chat answers follow direct-answer-first contract
- Separates facts, assumptions, caveats, and recommended next action
- Deep mode includes conclusion, reasoning, tradeoff, and next step
- Research mode labels whether claims came from live retrieval or model/context
- Long-context compression preserves active task, decisions, verified facts,
  key content, verification status, and open questions

SELF-UPGRADE 2.0 SYSTEM (Ultron-exclusive)
--------------------------------------------------------------------------------
Ultron can analyze and improve its own source code — with strict guardrails:

HOW IT WORKS:
1. Click “Upgrade” in the sidebar to open the Self-Upgrade panel
2. Choose an upgrade pack, quick task, backlog item, or custom improvement
3. The agent reads the codebase using code_search, read_file, list_directory
4. Each run is recorded with risk, impact, affected files, and validation needs
5. Before/after safety snapshots track pending previews, applied previews,
  rollbackable changes, and tool count
6. Proposes changes via preview_write (shows diff for user review)
7. User reviews diffs in the Preview panel and applies, discards, or rolls back
  applied file previews

GUARDRAILS (strictly enforced):
• Only permitted tools: read_file, list_directory, code_search, diff_files,
  lint_code, preview_write
• Direct write_file is BLOCKED — all changes go through preview/approval
• Only src/ and server/ directories may be modified
• Protected files: package.json, tsconfig*.json, node_modules, .env, dist/,
  .chat-history, *.jsonl, vite.config.ts, electron/
• Changes are minimal and surgical (no whole-file rewrites)
• Lint validation required after each proposed change
• Max 15 reasoning steps per upgrade session

Quick tasks built-in:
• Review codebase for improvement opportunities
• Fix TypeScript errors
• Optimize agent loop performance
• UI accessibility audit
• Security review
• Add analytics to agent

Upgrade packs built-in:
• Router Intelligence Pack
• Connector Pack
• Performance Pack
• UI Polish Pack
• Memory Pack

Endpoints: GET /api/self-upgrade, POST /api/self-upgrade (SSE),
           GET /api/previews/applied, POST /api/previews/:id/rollback

--------------------------------------------------------------------------------
SELF-HEALING SYSTEM
--------------------------------------------------------------------------------
Ultron can detect and propose fixes for its own code errors — with strict
guardrails so nothing ever changes without your explicit approval.

HOW IT WORKS:
1. Click "Heal" in the sidebar to open the Self-Healer panel
2. Click "Scan" — runs tsc --noEmit against the codebase (TypeScript errors only)
3. Each detected error shows with file, line, error code, and message
4. Click "Fix" on any issue to launch the Healing Agent
5. The agent reads the file, diagnoses the error, and proposes a minimal fix
6. The fix appears in the Preview panel (diff view) for your review
7. Click "Apply" to write it, or "Discard" to skip — nothing changes without you

GUARDRAILS:
• Healing agent is RESTRICTED: only read_file, code_search, lint_code, preview_write
• write_file is BLOCKED — all changes route through the preview/approval flow
• Rate limited: 1 scan per minute, 1 heal per 5 minutes (prevents loops)
• Max 6 reasoning steps per heal attempt
• Temperature 0.1: conservative, deterministic fixes
• Full audit log of every attempt (last 20 stored in memory)
• Agent will reply "CANNOT_HEAL: reason" if it can't fix safely

WHAT IT DETECTS:
• TypeScript compilation errors (TS2345, TS2322, TS2304, etc.)
• Type mismatches, missing properties, wrong function signatures
• Import errors, undefined variables, unused variables (warnings)
